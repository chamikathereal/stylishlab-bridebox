# 🚀 Professional Deployment Guide: Stylish Lab Bridebox (Full Version)

This guide provides a detailed, step-by-step walkthrough of the entire deployment process, including editor commands and security best practices.

## 🛠 PHASE 1: Server Preparation

### 1. Update & Basic Tools

```bash
# Update local package index and upgrade all packages
sudo apt update && sudo apt upgrade -y

# Install essential software
sudo apt install -y curl git wget build-essential ufw unattended-upgrades
```

### 2. Configure Swap Space (CRITICAL)

Prevents crashes during high memory tasks like building Next.js.

```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 3. Create Deployment User

```bash
sudo adduser deploy
# Enter a password and fill details or press Enter to skip

# Give sudo permissions (Allows running commands without password)
echo "deploy ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/deploy
```

## 🚀 PHASE 4: Application Deployment

**Switch to the deploy user first:**

```bash
su - deploy
```

### 1. Clone Project

```bash
git clone https://github.com/chamikathereal/stylishlab-bridebox.git
cd stylishlab-bridebox
```

### 2. Configure & Build Backend (Java)

```bash
cd ~/stylishlab-bridebox/stylishlab-bridebox-backend

# Update database credentials automatically
sed -i 's/spring.datasource.username=.*/spring.datasource.username=bridebox_user/' src/main/resources/application.properties
sed -i 's/spring.datasource.password=.*/spring.datasource.password=Ceegthereal@2003/' src/main/resources/application.properties

# Build the JAR file
chmod +x mvnw
./mvnw clean package -DskipTests
```

### 3. Configure & Build Frontend (Next.js)

**Step A: Update API URL**

```bash
# Navigate to the API config folder
cd ~/stylishlab-bridebox/stylishlab-bridebox-frontend/src/api

# Open the file using nano editor
nano axios-instance.ts
```

**Inside the Editor:**

1. Find the line: `baseURL: 'http://localhost:8080',`
2. Change it to: `baseURL: 'https://stylishlab-bridebox.duckdns.org/api',`
3. **Save and Exit:**
   - Press **Ctrl + O** then **Enter** to Save.
   - Press **Ctrl + X** to Exit.

**Step B: Install & Build**

```bash
cd ~/stylishlab-bridebox/stylishlab-bridebox-frontend
npm install
npm run build
```


**Paste this content (Right-click to paste):**

## 🌐 PHASE 6: Nginx & SSL

### 1. Firewall Setup

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw --force enable
```

### 2. Configure Nginx Proxy

```bash
sudo nano /etc/nginx/sites-available/bridebox
```

**Paste this content:**

```nginx
server {
    listen 80;
    server_name stylishlab-bridebox.duckdns.org;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    location /api {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**Save and Exit:** (Ctrl+O, Enter, Ctrl+X)

**Enable Config:**

```bash
sudo ln -s /etc/nginx/sites-available/bridebox /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx
```

### 3. SSL (HTTPS)

```bash
sudo certbot --nginx -d stylishlab-bridebox.duckdns.org
# Follow prompts: Enter Email -> Agree Terms (Y) -> Redirect (2)
```
